# Tasks

## 目的
- golang で標準入力 or option で受け取ったテキスト or markdown or pdf を翻訳する CLI を作る
- テキストはレイアウト無視
- markdown はフォーマット維持
- pdf はレイアウト維持
- 翻訳は gpt-oss 20b を使用し、`kirgizu:8080` の HTTP API を利用する

## タスク案
1. 要件確認
   - CLI 仕様（サブコマンド or フラグ、入出力、言語指定など）の確定
   - `kirgizu:8080` の API 仕様（エンドポイント、リクエスト/レスポンス形式、モデル名、認証の有無）確認
2. プロジェクト初期化
   - go.mod 作成、基本ディレクトリ構成
3. 翻訳クライアント実装
   - HTTP クライアント（タイムアウト、リトライ方針、ログ）
4. テキスト翻訳機能
   - stdin / ファイル / オプション入力
5. Markdown 翻訳機能
   - フォーマット保持（コードブロック、リンク、リストなど）
6. PDF 翻訳機能
   - レイアウト保持（フォント、位置情報）
7. CLI 実装
   - 引数解析、入出力、エラー処理
8. テスト
   - 最小限のユニットテスト + 必要に応じた統合テスト
9. ドキュメント
   - 使い方、例、開発メモ

## 確認したい事項
- CLI の具体仕様（例: `translate --from ja --to en --in file.md --out out.md` など）
- `kirgizu:8080` の API 仕様（エンドポイント・ペイロード・モデル指定の方法）
- PDF 処理に使ってよいライブラリ（指定がなければ候補を提案）
